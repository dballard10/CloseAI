# ClosedAI

ClosedAI is an evaluated privacy gate for external LLM consultation.

A trusted local orchestrator sees the raw prompt, detects sensitive entities,
creates a semantic abstraction, checks the abstraction for leakage, repairs it
when needed, and only then asks an external consultant for generic advice. The
trusted finalizer can use the original private context; the external consultant
only sees the sanitized prompt.

## Architecture

```text
Next.js TypeScript frontend
        ↓
Next.js API route: POST /api/run
        ↓
FastAPI Python service: POST /run
        ↓
Trusted local multi-agent pipeline
        ↓
Optional W&B Weave tracing and eval metadata
```

Pipeline:

```text
raw prompt
→ entity detection
→ semantic de-identification
→ checker
→ repair loop
→ utility evaluator
→ privacy gate
→ external consultant
→ trusted finalizer
```

## Run Locally

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
npm install
npm run dev
```

The app defaults to:

- Frontend: http://localhost:3000
- Python service: http://127.0.0.1:8000

If port `3000` is already in use:

```bash
npx concurrently -k -n python,next -c green,blue \
  ".venv/bin/uvicorn app.server:app --reload --port 8000" \
  "next dev -p 3001"
```

## API

```http
POST /api/run
```

```ts
type RunRequest = {
  rawPrompt: string;
  mode: "hr" | "legal" | "healthcare" | "education" | "general";
};
```

The response includes detected entities, initial and repaired sanitized prompts,
checker results, utility score, external consultant response, final trusted
answer, prompt versions, and Weave-compatible trace/eval metadata.

## LLM Mode

ClosedAI can run the trusted agents through W&B Inference or local Ollama
instead of the deterministic fallback. For the W&B-backed setup:

```bash
export CLOSEDAI_USE_LLM=1
export CLOSEDAI_INTERNAL_PROVIDER=wandb
export CLOSEDAI_INTERNAL_MODEL=OpenPipe/Qwen3-14B-Instruct
export WANDB_API_KEY=...
export WEAVE_PROJECT=closeai
```

That sends raw private prompts to W&B Inference for the trusted/internal
de-identification and checker calls. Use Ollama instead when you need those
trusted calls to stay on the machine:

```bash
export CLOSEDAI_USE_LLM=1
export CLOSEDAI_INTERNAL_PROVIDER=ollama
export OLLAMA_HOST=http://127.0.0.1:11434
export CLOSEDAI_LOCAL_MODEL=llama3.2:3b
export CLOSEDAI_LLM_STEPS=pipeline,finalize
export CLOSEDAI_UTILITY_RETRIES=1
```

By default the live app uses this LLM loop:

```text
question
→ one local LLM de-identification call
→ one local LLM check call for privacy and utility
→ maybe one more local LLM de-identification call with checker feedback
→ one final local LLM check call
```

The default web flow does not use the regex/NLP detector path. If the configured
trusted LLM is unavailable, the service falls back to the older deterministic
test path.

The final trusted answer is generated by the configured trusted/internal model
when `finalize` is included in `CLOSEDAI_LLM_STEPS`.

The external consultant can use OpenAI after the privacy gate passes:

```bash
export OPENAI_API_KEY=...
export CLOSEDAI_EXTERNAL_PROVIDER=openai
export CLOSEDAI_EXTERNAL_MODEL=gpt-5.5
```

`gpt-5.5` is the default external model. You can set
`CLOSEDAI_EXTERNAL_MODEL` to another OpenAI model your account exposes. If
`CLOSEDAI_EXTERNAL_PROVIDER` is blank, ClosedAI auto-selects OpenAI when
`OPENAI_API_KEY` is set, then W&B Inference when `WANDB_API_KEY` is set,
otherwise the mock consultant. Put real keys in a local `.env` file or shell
export, not in `.env.example`.

Do not commit real API keys. Use shell exports or a local `.env` file.

## Evaluation

ClosedAI includes a synthetic evaluation set for HR, legal, healthcare,
education, and general private-context scenarios.

```bash
.venv/bin/python eval/private_consult_evaluate.py
```

The eval reports direct leakage, preservation, checker accuracy, and repair
success. W&B Weave is optional at runtime; when `WANDB_API_KEY` or
`WANDB_API_KEY_FILE` is configured, the existing `@op` wrappers can emit live
traces. Without credentials, the app runs non-interactively and returns offline
trace metadata.
